## Feature Description

Two pure Dart CLI programs implementing zenoh pub/sub with shared memory (SHM). `counter_pub.dart` publishes an incrementing int64 counter via SHM zero-copy. `counter_sub.dart` subscribes and decodes the raw little-endian int64 payload. Both programs accept CLI flags mirroring zenoh-c conventions (-k, -e, -l, and -i for the publisher). A shared helper library (`lib/counter_codec.dart`) handles int64 encoding/decoding. This is the first template repo validating the zenoh-dart package.

## Requirements

### Functional Requirements
- [ ] `counter_pub.dart` publishes incrementing int64 values via SHM on a configurable key expression
- [ ] `counter_sub.dart` subscribes and decodes little-endian int64 payloads from any publisher
- [ ] Both programs parse CLI flags: `-k/--key`, `-e/--connect`, `-l/--listen`
- [ ] Publisher parses additional `-i/--interval` flag (milliseconds)
- [ ] Counter payload is raw little-endian int64 (8 bytes)
- [ ] SHM pipeline: allocGcDefragBlocking(8) -> write int64 -> toBytes -> putBytes
- [ ] Graceful SIGINT shutdown: close publisher/subscriber, provider, session
- [ ] Idempotent close for all resources

- [ ] Code passes `fvm dart analyze` with no errors
- [ ] All commands use `fvm dart` (bare `dart` NOT on PATH)
- [ ] No FFI mocking -- tests use real zenoh through native libraries
- [ ] Tests require `LD_LIBRARY_PATH` for native libraries

## Implementation Notes

### Architectural Decisions

- **Shared codec in lib/**: `counter_codec.dart` is extracted to `lib/` because both `counter_pub.dart` and `counter_sub.dart` need int64 encoding/decoding. This also makes the codec independently testable without zenoh dependencies.
- **Shared args in lib/**: `counter_args.dart` is extracted to `lib/` so both entrypoints share identical flag parsing logic. `parsePubArgs` extends `parseCommonArgs` with the `-i/--interval` flag.
- **Two-session TCP testing**: Following zenoh-dart's established pattern, integration tests use explicit TCP listen/connect on unique ports per test group (ports 17460-17463). Multicast scouting does not work within a single process.
- **No mocking**: Tests call real zenoh through `libzenoh_dart.so`. This requires `LD_LIBRARY_PATH` to be set.
- **FVM required**: All dart commands use `fvm dart` since bare `dart` is not on PATH.
- **package:zenoh must be added**: The current `pubspec.yaml` only has `args` and `test`. The zenoh git dependency must be added before Slice 3.
- **Timer.periodic delays**: Integration tests need `Future.delayed` for session link establishment (~1s) and routing propagation (~1s), matching the pattern in zenoh-dart's own tests.
- **Existing bin/zenoh_counter_dart.dart**: The scaffold template file should be removed and replaced by `bin/counter_pub.dart` and `bin/counter_sub.dart`.
- **allocGcDefragBlocking with oversized request**: Verified against zenoh-dart C shim -- returns null (not block forever) when size exceeds pool capacity. The blocking behavior only applies to transient space shortages resolvable by GC/defrag.
